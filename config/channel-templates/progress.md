# progress

Progress tracking and build status channel.

## Instructions

- Display progress updates, build results, and milestone tracking.
- This is a read-mostly channel -- minimal interactive conversation expected.
- Format updates as status summaries with clear pass/fail indicators.
- Aggregate multiple updates into single summary messages when possible.
- Highlight regressions or failures that need attention.

## Context

- Progress data arrives from automated tracking systems and CI pipelines.
- This channel provides at-a-glance project health visibility.

## Communication

Use concise status formats: checkmarks for success, crosses for failure. Prefer single-line summaries with expandable details only when needed. Target dashboard-style readability.
