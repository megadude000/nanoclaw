    **Add to `src/discord-server-manager.ts`:**

    1. Define zod schemas for config validation (ServerConfigSchema, CategoryConfigSchema, ChannelConfigSchema) at module level.

    2. Add `bootstrap` to the `handleAction` dispatch.

    3. Implement private `bootstrap()` method per D-05:
       - Read `config/discord-server.json` via `fs.readFileSync` with path resolved relative to project root (use `import.meta.url` or `__dirname` to find project root, or accept config path in constructor/params)
       - Validate with zod schema. Return error if invalid.
       - Call `guild.channels.fetch()` to get fresh channel list (not just cache -- per research Pitfall 1)
       - For each category in config:
         - Find existing category by name (case-insensitive comparison, `ChannelType.GuildCategory`) per research Pitfall 3
         - If not found, create it via `guild.channels.create({ name, type: ChannelType.GuildCategory })`
         - For each channel in category:
           - Find existing channel by name + parentId match
           - If not found, create via `guild.channels.create({ name, type: ChannelType.GuildText, parent: categoryId, topic })`
           - Track created vs skipped
       - Return `{ success: true, created: [...], skipped: [...], total_created: N, total_skipped: N }`
       - Wrap each individual create in try/catch so one failure does not abort the entire bootstrap per D-11 and research anti-pattern guidance. Collect errors and continue.

    **Add tests to `src/discord-server-manager.test.ts`:**
    - bootstrap with empty guild: creates all 4 categories and 8 channels
    - bootstrap with existing structure: creates nothing (idempotent) -- mock guild.channels.fetch to return all expected channels
    - bootstrap with partial structure: creates only missing items
    - bootstrap with invalid config: returns error
    - bootstrap with one channel creation failing: continues creating others, reports partial result
