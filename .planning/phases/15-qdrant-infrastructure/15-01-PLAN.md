Create `scripts/qdrant-setup.sh` (chmod +x):
- Pull `qdrant/qdrant:latest` if not already present
- Run container named `nanoclaw-qdrant` with:
  - `--restart=unless-stopped` (Docker-level restart as safety net)
  - `-v $(pwd)/data/qdrant:/qdrant/storage:z` (bind-mount per D-01, `:z` for SELinux compat)
  - `-p 127.0.0.1:6333:6333` (localhost-only binding -- no external exposure)
  - `-p 127.0.0.1:6334:6334` (gRPC port, localhost-only)
  - `--name nanoclaw-qdrant`
- Before starting, check if container already exists: `docker ps -a --filter name=nanoclaw-qdrant --format '{{.Names}}'`. If exists and running, exit 0. If exists but stopped, `docker start nanoclaw-qdrant`. If does not exist, `docker run -d ...`.
- Create `data/qdrant/` directory if missing: `mkdir -p data/qdrant`

Create `.config/systemd/qdrant.service` (a template file stored in repo, installed by copying to `~/.config/systemd/user/`):
```ini
[Unit]
Description=Qdrant Vector Database for NanoClaw Cortex
After=network.target docker.service
Requires=docker.service

[Service]
Type=oneshot
RemainAfterExit=yes
WorkingDirectory=/home/andrii-panasenko/nanoclaw
ExecStart=/home/andrii-panasenko/nanoclaw/scripts/qdrant-setup.sh
ExecStop=/usr/bin/docker stop nanoclaw-qdrant
TimeoutStartSec=120

[Install]
WantedBy=default.target
```

2. Copy `.config/systemd/qdrant.service` to `~/.config/systemd/user/qdrant.service`

4. `systemctl --user enable qdrant.service`

6. Verify: `docker ps --filter name=nanoclaw-qdrant --format '{{.Status}}'` shows "Up"

Note: The `.config/systemd/qdrant.service` file in the repo serves as the canonical template. The actual installed path is `~/.config/systemd/user/qdrant.service`.

  <action>
Create `scripts/qdrant-collection.sh` (chmod +x):

Uses Qdrant REST API (curl) to create the `cortex-entries` collection and payload indexes.

Step 1 -- Create collection:
```bash
curl -sf -X PUT http://localhost:6333/collections/cortex-entries \
  -H 'Content-Type: application/json' \
  -d '{
    "vectors": {
      "size": 1536,
      "distance": "Cosine"
    },
    "hnsw_config": {
      "m": 16,
      "ef_construct": 100
    },
    "optimizers_config": {
      "indexing_threshold": 100
    }
  }'
```

Vector size 1536 matches OpenAI text-embedding-3-small (from REQUIREMENTS.md EMBED-01).
HNSW m=16, ef_construct=100 are standard defaults good for collections up to ~100K vectors.
indexing_threshold=100 means HNSW index builds after 100 points (good for small collections).

Step 2 -- Create payload indexes (one curl per field):

- `project` (keyword): filters search by project name
  ```bash
  curl -sf -X PUT http://localhost:6333/collections/cortex-entries/index \
    -H 'Content-Type: application/json' \
    -d '{"field_name": "project", "field_schema": "keyword"}'
  ```
- `cortex_level` (keyword): filters by L10-L50
  ```bash
  curl -sf -X PUT http://localhost:6333/collections/cortex-entries/index \
    -H 'Content-Type: application/json' \
    -d '{"field_name": "cortex_level", "field_schema": "keyword"}'
  ```
- `domain` (keyword): filters by domain scope
  ```bash
  curl -sf -X PUT http://localhost:6333/collections/cortex-entries/index \
    -H 'Content-Type: application/json' \
    -d '{"field_name": "domain", "field_schema": "keyword"}'
  ```
- `status` (keyword): filters by entry status (active, stale, archived)
  ```bash
  curl -sf -X PUT http://localhost:6333/collections/cortex-entries/index \
    -H 'Content-Type: application/json' \
    -d '{"field_name": "status", "field_schema": "keyword"}'
  ```

The script must be idempotent:
- Check if collection exists first: `curl -sf http://localhost:6333/collections/cortex-entries`. If 200, skip creation.
- Payload index creation is inherently idempotent in Qdrant (re-creating an existing index is a no-op).

After creating the script:

3. Verify collection: `curl -s http://localhost:6333/collections/cortex-entries | jq '.result.config.params.vectors.size'` should return 1536
4. Verify indexes: `curl -s http://localhost:6333/collections/cortex-entries | jq '.result.payload_schema'` should show project, cortex_level, domain, status as keyword indexes
