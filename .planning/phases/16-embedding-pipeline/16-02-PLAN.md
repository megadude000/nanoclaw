  <behavior>
    - startCortexWatcher() starts fs.watch on the given cortex directory with recursive:true
    - Only .md file changes are tracked (other extensions ignored)
    - Changes are collected in a Set and processed after 10 minutes (600000ms) of no new changes (per D-01)
    - Files in the "in-flight" set (being written by embedder) are ignored in the watcher callback
    - stopCortexWatcher() clears the debounce timer, closes the watcher, returns cleanly
    - When the batch callback fires, it calls embedEntry for each changed file and logs results
    - If Qdrant health check fails at start, watcher logs warning and does not start (graceful degradation)
    - If OPENAI_API_KEY is missing, watcher logs warning and does not start
  </behavior>
  <action>
    Create `src/cortex/watcher.ts`:

    1. **Module-level state:**
       - `let watcher: FSWatcher | null = null`
       - `let debounceTimer: ReturnType<typeof setTimeout> | null = null`
       - `const changedFiles = new Set<string>()`
       - `const inFlightFiles = new Set<string>()` -- files currently being written by the embedder (self-trigger prevention per Pitfall 1 from RESEARCH.md)

    2. **startCortexWatcher(cortexDir: string): Promise<void>** -- Per D-04, called from main() in index.ts:
       a. Check OPENAI_API_KEY via readEnvFile. If missing, log warning "Cortex watcher disabled: OPENAI_API_KEY not found in .env" and return.
       b. Create OpenAI client via createOpenAIClient().
       c. Create Qdrant client via createQdrantClient().
       d. Call checkQdrantHealth(qdrant). If false, log warning "Cortex watcher disabled: Qdrant not reachable at localhost:6333" and return.
       e. Start fs.watch(cortexDir, { recursive: true }) with callback:
          - Filter: ignore if no filename, not .md extension, or filename is in inFlightFiles
          - Add full path (path.join(cortexDir, filename)) to changedFiles Set
          - Clear existing debounceTimer
          - Set new debounceTimer for DEBOUNCE_MS (600000 = 10 minutes per D-01)
          - On timer fire: copy changedFiles to array, clear set, call processBatch()
       f. Log "Cortex watcher started" with cortexDir and debounce interval.

    3. **processBatch(files, openai, qdrant):** -- Internal function:
       a. Log batch start with file count.
       b. For each file, add to inFlightFiles before embedding, remove after.
       c. Call embedEntry(file, openai, qdrant) for each file sequentially.
       d. Log summary: { embedded: N, skipped: N, errors: N }.

    4. **stopCortexWatcher(): void** -- Cleanup for graceful shutdown:
       a. If debounceTimer, clearTimeout.
       b. If watcher, close it.
       c. Clear changedFiles and inFlightFiles sets.
       d. Set watcher and debounceTimer to null.

    5. **Export inFlightFiles set** (as `getInFlightFiles()` for testing only).

    Create `src/cortex/watcher.test.ts`:
    - Mock fs.watch to simulate file change events
    - Mock embedEntry to track calls
    - Test: .md files are added to changedFiles, .txt files are ignored
    - Test: inFlightFiles prevents re-processing
    - Test: stopCortexWatcher clears state
    - Use vi.useFakeTimers() to test debounce behavior without waiting 10 minutes
