  <action>
    Create `src/health-monitor.ts`:

    Types:
    ```typescript
    type ServiceState = 'up' | 'down' | 'unknown';
    type SendEmbedFn = (embed: EmbedBuilder) => Promise<void>;
    interface ServiceConfig { name: string; command: string; }
    ```

    Helper `execAsync(cmd: string, timeoutMs = 5000): Promise<{ stdout: string; stderr: string }>`:
    - Wraps child_process.exec in a Promise
    - Sets `{ timeout: timeoutMs }` option
    - On error, reject with error (which has stdout/stderr on it)

    Helper `loadState(services: string[]): Map<string, ServiceState>`:
    - Try read `path.join(DATA_DIR, 'health-state.json')` with fs.readFileSync
    - Parse JSON, restore only keys that match current service names
    - On any error, return map with all services set to 'unknown'

    Helper `saveState(stateMap: Map<string, ServiceState>): void`:
    - Write `Object.fromEntries(stateMap)` to `path.join(DATA_DIR, 'health-state.json')`
    - Use fs.writeFileSync, catch and log errors

    Helper `buildServiceConfigs(): ServiceConfig[]`:
    - Read `HEALTH_MONITOR_SERVICES` and `CLOUDFLARE_TUNNEL_NAME` from readEnvFile
    - Also check process.env for both
    - Parse comma-separated services (default: 'nanoclaw')
    - For each service: command = `systemctl --user is-active ${service}`
    - If CLOUDFLARE_TUNNEL_NAME is set, add config: name='cloudflared', command='systemctl is-active cloudflared' (no --user)
    - Return array of ServiceConfig

    Main `checkService(svc: ServiceConfig): Promise<ServiceState>`:
    - Run execAsync(svc.command)
    - If stdout.trim() === 'active', return 'up', else return 'down'
    - On exec error (non-zero exit), return 'down'

    Main `async function runHealthCheck(services, stateMap, sendEmbed)`:
    - For each service, call checkService
    - Compare with stateMap previous state
    - If transition is unknown->down, up->down: call sendEmbed(buildDownEmbed(name, stderr))
    - If transition is down->up: call sendEmbed(buildUpEmbed(name))
    - If transition is unknown->up: do NOT post (suppress startup spam)
    - Update stateMap, call saveState if any transition occurred
    - Log transitions with logger.info

    Main `async function runHeartbeat(stateMap, services, sendEmbed)`:
    - If ALL service states in stateMap are 'up': sendEmbed(buildHeartbeatEmbed(service names))
    - Otherwise skip

    Export `startHealthMonitor(sendEmbed: SendEmbedFn): () => void`:
    - Call buildServiceConfigs() to get services
    - If services.length === 0, log warning and return no-op cleanup
    - Call loadState(service names) to init stateMap
    - Set checkIntervalMs from HEALTH_CHECK_INTERVAL_MS env (default 60000)
    - Set heartbeatIntervalMs = 1800000 (30 minutes)
    - setTimeout for first poll at 10000ms, then setInterval at checkIntervalMs
    - setInterval for heartbeat at heartbeatIntervalMs
    - Log startup: `logger.info({ services: services.map(s => s.name), checkIntervalMs }, 'Health monitor started')`
    - Return cleanup function that clears all timers (clearInterval + clearTimeout)

    Create `src/health-monitor.test.ts`:
    - Mock child_process.exec using vi.mock('child_process')
    - Mock fs for state persistence tests
    - Test state transitions: unknown->up (no embed), unknown->down (embed), up->down (embed), down->up (embed)
    - Test heartbeat: only fires when all services up
    - Test buildServiceConfigs: default 'nanoclaw', custom list, cloudflared added when env set
    - Test loadState: reads from file, handles missing file
    - Test saveState: writes JSON
    - Test exec timeout is set to 5000
    - Follow vitest patterns from agent-status-embeds.test.ts
