    Create `src/webhook-router.ts` per D-10 with:

    1. **Zod schema** (per D-03, D-05, D-06):
       ```typescript
       const TargetSchema = z.object({
         platform: z.enum(['telegram', 'discord']),
         jid: z.string(),
       });
       const WebhookRouteSchema = z.object({
         targets: z.array(TargetSchema).min(1),
       });
       const RoutingConfigSchema = z.record(z.string(), WebhookRouteSchema);
       ```

    2. **RouteTarget interface** (per D-10):
       ```typescript
       export interface RouteTarget {
         jid: string;
         group: RegisteredGroup;
       }
       ```

    3. **resolveTargets function** (per D-10, D-11, D-12):
       - Accepts `webhookType: string` and `groups: Record<string, RegisteredGroup>`
       - CONFIG_PATH resolves to `config/routing.json` relative to project root (use `resolve(__dirname, '..', 'config', 'routing.json')` matching discord-server-manager.ts pattern)
       - Per D-02: reads config fresh every call with `readFileSync` (no caching)
       - Per D-12: if config file missing, returns mainJid fallback
       - Per D-12: if webhookType not in config, returns mainJid fallback
       - Per D-03: validates config with Zod, falls back on validation error
       - Resolves each target JID against groups map; skips unregistered JIDs with `logger.warn`
       - Per D-09: logs routing decisions at info level, failures at warn level
       - If no targets resolve, falls back to mainJid
       - Returns `RouteTarget[]`

    4. **mainFallback helper** (internal):
       - Finds first group where `g.isMain === true`
       - Returns `[{ jid, group }]` or `[]` if no main group
