## Tech Debt

### Phase 20: Lore Protocol — Missing Production Trigger

**Items:**
1. `lore-parser.ts` and `lore-mining.ts` are implemented, unit-tested (13 tests pass), and internally correct — but have no production call site.
2. `cortex_reconcile` IPC handler calls only `runReconciliation()`, which does not invoke lore mining.
3. LORE-03 (lore atoms searchable via cortex_search) cannot be achieved at runtime until a production trigger is added.
4. **Workaround:** Manual invocation via `npx tsx scripts/...` or adding a new IPC handler `cortex_lore_index`.

**Suggested fix:** Wire `mineLoreFromHistory()` into either:
- `runReconciliation()` in `reconciler.ts` (simplest — run on every Night Shift cycle)
- A new nightly scheduled task in `task-scheduler.ts`
- A new IPC handler `cortex_lore_mine` triggered by Night Shift agent

**Severity:** Medium — the missing trigger means git decision context isn't automatically indexed, but Cortex is otherwise fully operational.
