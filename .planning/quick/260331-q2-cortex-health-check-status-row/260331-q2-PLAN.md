# Plan: Cortex Health Check Status Row

## Goal
Add `✅ cortex` row to the "💚 All systems operational" heartbeat embed.
The check verifies the Qdrant Docker container is running.

## Context
- `src/health-monitor.ts` — `buildServiceConfigs()` builds the service list
- `buildHeartbeatEmbed(serviceNames)` renders `✅ {name}` per service
- Cloudflared uses the same pattern: env var gates inclusion, custom command

## Tasks

### Task 1 — Add cortex check to `buildServiceConfigs()` in `src/health-monitor.ts`

After the cloudflared block (line ~142–147), add:

```ts
// Cortex (Qdrant): check via docker inspect
const qdrantContainer =
  process.env.QDRANT_CONTAINER_NAME ??
  envValues.QDRANT_CONTAINER_NAME ??
  'nanoclaw-qdrant';

configs.push({
  name: 'cortex',
  command: `docker inspect --format='{{if .State.Running}}active{{end}}' ${qdrantContainer}`,
});
```

Read `QDRANT_CONTAINER_NAME` in `readEnvFile` call at top of function — add it to the array alongside `HEALTH_MONITOR_SERVICES` and `CLOUDFLARE_TUNNEL_NAME`.

Default is always-on: always adds cortex unless `QDRANT_CONTAINER_NAME` is explicitly set to empty string.

### Task 2 — Update tests in `src/health-monitor.test.ts`

In `beforeEach`/`afterEach`, add `delete process.env.QDRANT_CONTAINER_NAME`.

Add tests to `describe('buildServiceConfigs')`:

1. `'always includes cortex service using default container name'` — no env set, expect cortex entry with `docker inspect` command containing `nanoclaw-qdrant`
2. `'uses QDRANT_CONTAINER_NAME env var for cortex check'` — set `QDRANT_CONTAINER_NAME=my-qdrant`, expect command contains `my-qdrant`
3. `'skips cortex check when QDRANT_CONTAINER_NAME is empty string'` — set `QDRANT_CONTAINER_NAME=''`, expect no cortex entry

## Must-haves
- `✅ cortex` appears in heartbeat embed when Qdrant container is running
- Uses `docker inspect --format='{{if .State.Running}}active{{end}}'` — outputs `active` when healthy
- `QDRANT_CONTAINER_NAME` env var overrides default `nanoclaw-qdrant`
- Empty string disables the check
- All existing tests still pass
- New tests cover the three scenarios above
